## 启动步骤

### 后端启动

```bash
cd backend
go run main.go
```

后端服务将在 `http://localhost:8080` 启动

### 前端启动

#### Admin Platform
```bash
cd frontend/admin
npm run dev
```

Admin平台将在 `http://localhost:3000` 启动

#### Arena Platform
```bash
cd frontend/arena
npm run dev
```

Arena平台将在 `http://localhost:3001` 启动

## 注意事项

1. **数据库连接**：确保MySQL服务已启动，并且数据库已创建
2. **配置文件**：确保 `backend/config.yaml` 中的数据库配置正确
3. **端口占用**：确保8080、3000、3001端口未被占用
4. **CORS配置**：前端地址已配置在CORS允许列表中

## 前端TypeScript类型错误

前端存在一些TypeScript类型错误，主要是API响应类型的问题。这些错误不影响运行时功能，但需要在后续开发中修复：

### Admin Platform
- API响应类型需要明确定义
- 部分未使用的变量需要清理

### Arena Platform  
- API响应类型需要明确定义
- 部分未使用的变量需要清理
- Space组件导入缺失

**注意**：这些是类型检查错误，不影响实际运行。可以通过以下方式临时解决：
1. 在开发模式下运行（`npm run dev`）不会阻止启动
2. 或者暂时禁用TypeScript严格检查
