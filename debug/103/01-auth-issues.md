## 未修复的问题

### 1. 查看个人信息测试失败
**测试用例**：`tests/01-auth.spec.ts:101` - 查看个人信息

**错误信息**：
```
Error: 登录后未跳转，当前URL: http://localhost:3000/login
```

**问题描述**：
- 登录操作执行后，页面仍然停留在登录页
- 从错误上下文看，邮箱和密码已填写，登录按钮处于active状态
- 但页面没有跳转到 `/profile` 页面

**可能原因**：
- 登录API请求可能失败但没有显示错误消息
- 导航逻辑可能有问题
- 可能需要更长的等待时间

**建议修复方向**：
- 检查登录API响应
- 增加登录后的等待时间
- 检查路由跳转逻辑

### 2. 修改个人信息测试失败
**测试用例**：`tests/01-auth.spec.ts:111` - 修改个人信息

**错误信息**：
```
TimeoutError: page.waitForSelector: Timeout 5000ms exceeded.
等待成功消息超时
```

**问题描述**：
- 个人信息修改操作可能已经完成（从错误上下文看，页面已显示新姓名）
- 但成功消息没有正确显示或选择器无法匹配

**可能原因**：
- Ant Design 消息组件的选择器可能不准确
- 消息可能已经显示但选择器匹配失败
- 消息显示时间过短

**建议修复方向**：
- 调整消息选择器，使用更通用的选择器
- 检查消息是否真的显示
- 增加消息等待的容错性
