# IDEA-401
我想制作一个Hackathon比赛平台项目，角色有主办方、参赛者和赞助商这三方，我总共想分成若干期完成这个项目
主办方和赞助商共用一个平台Hackathon Admin Platform，参赛者使用另一个平台Hackathon Arena Platform
## 第四期
第四期完成如下内容，全面支持活动上链
### 1. 活动信息上链
#### Admin平台
##### 主办方
+ 主办方增删改查活动时，链上会同步记录，活动阶段信息不需要同步至链上
#### Arena平台
##### 主办方
+ 在签到阶段，参赛者签到后，主办方会给参赛者发放NFT
##### 参赛者
+ 在签到阶段，参赛者的签到信息会同步到链上，签到需要有防重校验
+ 在投票阶段，参赛者的投票信息会同步到链上，投票操作可撤销
+ 在活动中，参赛者可以在系统中验证当前活动信息的真实性，包括活动本身和投票记录
+ 活动后，在过往活动列表中，任何游客可以在系统中验证当前活动信息的真实性，包括活动本身和投票记录
### 2. 资金链路上链
所有资金链路可在Admin平台和Arena平台查询
#### Admin平台
##### 主办方
+ 主办方在创建活动时，会创建智能合约托管奖金，如果主办方钱包中的金额不够，活动则会创建失败
+ 在活动比赛结果公布后，奖金会根据比赛结果分发给对应的队伍
##### 赞助商
###### 长期赞助商
+ 赞助商在申请活动时，需要输入赞助金额，并会创建智能合约托管奖金，如果赞助商钱包中的金额不够，则会申请失败
+ 主办方通过后，赞助金额会转入主办方钱包；主办方驳回赞助商对于某个活动的申请后，智能合约托管奖金会原路返回给赞助商
###### 指定活动赞助商
+ 赞助商在赞助某活动时，需要输入赞助金额，并会创建智能合约托管奖金，如果赞助商钱包中的金额不够，则会申请失败
+ 主办方驳回赞助商对于某个活动的申请后，智能合约托管奖金会原路返回给赞助商
+ 赞助商赞助的资金分发方式会等比例复制活动本身的分发方式
#### Arena平台
##### 参赛者
+ 在活动比赛结果公布后，奖金会根据比赛结果分发给对应的队伍
+ 在组队时，队长可以设置奖金分成比例，如取得名次，奖金会根据队长设置的奖金分成比例再次自动分配给队员
